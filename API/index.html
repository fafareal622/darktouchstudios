API Documentation

Welcome to the API documentation for the Sample API!

Introduction

The Sample API allows you to interact with the fictional platform to retrieve data, manage resources, and perform various actions. Below are details of the available endpoints and their usage.

Base URL

All API requests are made to the following base URL:

https://darktouchstudios.pages.dev/api/v1

Authentication

All requests to the API must include an API key in the request header. You can obtain your API key by signing up on the platform.

Endpoints

GET /users

Retrieve a list of all users on the platform.

Request Example:

GET https://darktouchstudios.pages.dev/api/v1/users

Response Example:

[
    {
        "id": 1,
        "name": "John Doe",
        "email": "johndoe@example.com"
    },
    {
        "id": 2,
        "name": "Jane Smith",
        "email": "janesmith@example.com"
    }
]

POST /users

Create a new user in the system.

Request Example:

POST https://darktouchstudios.pages.dev/api/v1/users

Request Body:

{
    "name": "Alice Cooper",
    "email": "alicecooper@example.com"
}

Response Example:

{
    "id": 3,
    "name": "Alice Cooper",
    "email": "alicecooper@example.com"
}

GET /users/{id}

Retrieve details of a specific user by their ID.

Request Example:

GET https://darktouchstudios.pages.dev/api/v1/users/1

Response Example:

{
    "id": 1,
    "name": "John Doe",
    "email": "johndoe@example.com"
}

Error Handling

If a request is invalid, the API will return an error response. Here are common error codes:

  • 400: Bad Request - The request could not be understood by the server due to malformed syntax.
  • 401: Unauthorized - The request requires user authentication.
  • 404: Not Found - The requested resource could not be found.
  • 500: Internal Server Error - An error occurred on the server.

Rate Limiting

The API has a rate limit of 1000 requests per hour. If you exceed this limit, you will receive a 429 Too Many Requests response until the limit resets.
